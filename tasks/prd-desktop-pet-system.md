# PRD: 桌面宠物系统 (Desktop Pet System)

## 1. Introduction / Overview

构建一个基于 Electron 的跨平台桌面宠物应用。宠物以矢量/卡通动画形式呈现在桌面之上，可在整个屏幕范围内自由活动，并能与窗口边缘、任务栏等系统元素发生交互。系统采用混合行为模式：宠物在无人干预时按状态机自主切换待机/行走/跳跃等行为；当用户与之交互（点击、拖拽）时，宠物给出对应反应。

为了便于后续持续扩展（例如新增"睡觉"、"玩耍"、"喂食"等行为），系统在架构上采用**插件式行为模块**设计——每个行为是一个独立、可注册的模块，遵循统一接口，通过状态机进行调度。

本 PRD 描述 v1.0 的核心目标与可交付范围。

---

## 2. Goals

- 提供一个常驻桌面、视觉友好且不干扰正常工作的虚拟陪伴角色
- 实现稳定的基础行为循环：待机 / 行走 / 跳跃 / 被拖拽 / 坠落
- 支持鼠标点击、拖拽等基础用户交互，并产生符合直觉的响应
- 宠物可感知并与屏幕边缘、任务栏等环境元素交互
- 行为系统采用插件式架构，便于后续无侵入地新增行为
- 在主流 Windows / macOS 系统上稳定运行，CPU/内存占用低

---

## 3. User Stories

> 每个 User Story 应在一次专注会话中可完成。涉及 UI 的 Story 必须使用 dev-browser 技能在浏览器/Electron 窗口中验证。

### US-001: 搭建 Electron 项目骨架

**Description:** 作为开发者，我需要一个具备透明、置顶、无边框窗口能力的 Electron 项目骨架，作为后续所有宠物功能的载体。

**Acceptance Criteria:**
- [ ] 使用 Electron + TypeScript 初始化项目（含 `package.json`、`tsconfig.json`、构建脚本）
- [ ] 主窗口配置为：透明背景、无边框、永远置顶、不在任务栏显示
- [ ] 窗口大小覆盖整个主屏幕（用于自由移动），但鼠标穿透在非宠物区域生效
- [ ] 项目可通过 `npm run dev` 启动并显示一个测试占位图形
- [ ] Typecheck / lint 通过
- [ ] 在 Electron 窗口中验证（dev-browser skill）

---

### US-002: 渲染宠物精灵与基础动画框架

**Description:** 作为用户，我希望看到一个有动画感的卡通形象出现在桌面上，作为后续所有行为的视觉基础。

**Acceptance Criteria:**
- [ ] 使用 SVG 或 Lottie 渲染宠物角色
- [ ] 实现动画播放器组件：支持加载、播放、循环、切换动画片段
- [ ] 至少内置一个"待机呼吸"动画作为默认表现
- [ ] 动画切换无明显闪烁或位置跳变
- [ ] Typecheck 通过
- [ ] 在 Electron 窗口中验证（dev-browser skill）

---

### US-003: 行为状态机与插件式行为接口

**Description:** 作为开发者，我需要一个可扩展的行为状态机，让每个行为（待机、行走、跳跃等）作为独立模块注册进来，由状态机统一调度。

**Acceptance Criteria:**
- [ ] 定义 `Behavior` 接口：`id`、`enter()`、`update(dt)`、`exit()`、`canTransitionTo(nextId)`、`priority`
- [ ] 实现 `BehaviorRegistry`：支持注册 / 注销行为模块
- [ ] 实现 `BehaviorStateMachine`：管理当前行为、处理切换、提供 `requestTransition(id, reason)` API
- [ ] 提供事件总线 (`EventBus`)，行为模块间通过事件解耦通信（如 `mouse:down`, `edge:hit`）
- [ ] 包含至少一个空壳示例行为，证明注册机制可用
- [ ] 单元测试覆盖：状态切换、优先级抢占、非法切换被拒绝
- [ ] Typecheck 通过

---

### US-004: 待机行为（Idle）

**Description:** 作为用户，我希望宠物在没有任何事情发生时呈现自然的待机状态，包括偶发小动作。

**Acceptance Criteria:**
- [ ] 实现 `IdleBehavior`，作为默认行为
- [ ] 播放呼吸/眨眼等循环动画
- [ ] 每隔 5–15 秒（随机）触发一次小动作（如打哈欠、看向四周）
- [ ] 待机期间宠物位置保持不变
- [ ] 可被更高优先级行为（如拖拽、行走）打断
- [ ] 在 Electron 窗口中验证（dev-browser skill）

---

### US-005: 行走行为（Walk）

**Description:** 作为用户，我希望宠物会自己在屏幕上四处走动，看起来"活着"。

**Acceptance Criteria:**
- [ ] 实现 `WalkBehavior`：选择一个随机目标点，沿直线匀速移动
- [ ] 行走速度可配置（默认 60–120 px/s）
- [ ] 根据移动方向自动翻转精灵朝向
- [ ] 播放对应行走动画
- [ ] 到达目标点后切回 `IdleBehavior`
- [ ] 移动过程中可被拖拽等高优先级行为打断
- [ ] 在 Electron 窗口中验证（dev-browser skill）

---

### US-006: 跳跃行为（Jump）

**Description:** 作为用户，我希望宠物偶尔自发跳跃，让画面更生动。

**Acceptance Criteria:**
- [ ] 实现 `JumpBehavior`：执行抛物线轨迹位移（起跳→上升→下落→落地）
- [ ] 起跳/落地播放对应动画片段
- [ ] 跳跃高度与水平距离可配置
- [ ] 跳跃完成后切回 `IdleBehavior` 或 `WalkBehavior`
- [ ] 在 Electron 窗口中验证（dev-browser skill）

---

### US-007: 拖拽行为（Drag）

**Description:** 作为用户，我希望可以用鼠标抓起宠物拖到任意位置。

**Acceptance Criteria:**
- [ ] 鼠标在宠物身上按下并移动时进入 `DragBehavior`（最高优先级，可打断任何其他行为）
- [ ] 宠物精灵跟随鼠标移动，无明显延迟
- [ ] 播放"被抓起"动画（如惊讶/挣扎）
- [ ] 鼠标松开时根据释放速度决定后续：低速 → `Idle`；高速 → `FallBehavior`
- [ ] 拖拽期间禁用自动行为切换
- [ ] 在 Electron 窗口中验证（dev-browser skill）

---

### US-008: 坠落行为（Fall）

**Description:** 作为用户，当我把宠物从高处松开后，希望它会自然地下落而不是悬空。

**Acceptance Criteria:**
- [ ] 实现 `FallBehavior`：施加重力加速度，竖直下落
- [ ] 落到屏幕底部边缘 / 任务栏顶部时停止
- [ ] 落地时播放"砸地"动画并轻微弹跳
- [ ] 落地后切回 `IdleBehavior`
- [ ] 在 Electron 窗口中验证（dev-browser skill）

---

### US-009: 屏幕边缘与任务栏感知

**Description:** 作为用户，我希望宠物能"踩"在任务栏上行走，并在碰到屏幕边缘时不消失到屏幕外。

**Acceptance Criteria:**
- [ ] 通过 Electron API 获取主屏幕工作区尺寸与任务栏位置
- [ ] 行走/坠落时以工作区底部为"地面"基准
- [ ] 触碰左右边缘时反弹或转身（行为可配置）
- [ ] 任务栏位置变化时（用户改变任务栏位置/隐藏）自动更新地面参数
- [ ] 在 Electron 窗口中验证（dev-browser skill）

---

### US-010: 用户交互响应（点击）

**Description:** 作为用户，我希望点击宠物时它会有反馈，让我感受到陪伴感。

**Acceptance Criteria:**
- [ ] 单击宠物：播放一次"开心/被摸"动画后回到原行为
- [ ] 双击宠物：触发一次跳跃
- [ ] 右键宠物：弹出菜单（含"退出"、"隐藏 30 分钟"、"设置"占位项）
- [ ] 在 Electron 窗口中验证（dev-browser skill）

---

### US-011: 自动行为调度器

**Description:** 作为用户，我希望宠物在没有交互时自己在 待机 / 行走 / 跳跃 之间自然切换。

**Acceptance Criteria:**
- [ ] 实现 `AutoScheduler`：在用户长时间无交互时按权重随机选择下一行为
- [ ] 行为权重与冷却时间可配置
- [ ] 用户交互（点击、拖拽）发生时调度器暂停 N 秒，避免抢断
- [ ] 调度器作为独立模块，与具体行为解耦
- [ ] 单元测试覆盖：权重分布、冷却生效、交互后暂停
- [ ] 在 Electron 窗口中验证（dev-browser skill）

---

### US-012: 配置文件与持久化

**Description:** 作为用户，我希望我对宠物的偏好（如行走速度、是否启用某行为）能被保存。

**Acceptance Criteria:**
- [ ] 在用户数据目录下保存 `config.json`
- [ ] 暴露 `ConfigStore` API：`get(key)` / `set(key, value)` / 变更事件订阅
- [ ] 行为模块通过 `ConfigStore` 读取自身参数
- [ ] 默认配置在首次启动时生成
- [ ] Typecheck 通过

---

### US-013: 性能与稳定性基线

**Description:** 作为用户，我希望宠物常驻桌面但不影响我的工作。

**Acceptance Criteria:**
- [ ] 空闲状态下 CPU 占用 < 2%（在中端笔记本上测量）
- [ ] 内存占用 < 200 MB
- [ ] 切换前后台、锁屏唤醒后宠物状态正确恢复
- [ ] 连续运行 8 小时无崩溃、无内存泄漏（通过本地观测验证）
- [ ] 在 Electron 窗口中验证（dev-browser skill）

## 4. Functional Requirements

- **FR-1**：应用以 Electron + TypeScript 构建，主窗口为透明、无边框、置顶。
- **FR-2**：宠物视觉采用矢量/卡通风格（SVG 或 Lottie），支持帧动画切换。
- **FR-3**：宠物可在整个主屏幕工作区内自由移动，地面以任务栏顶部为基准。
- **FR-4**：实现行为状态机，所有行为以插件形式注册到 `BehaviorRegistry`。
- **FR-5**：必须实现的内置行为：`Idle`、`Walk`、`Jump`、`Drag`、`Fall`。
- **FR-6**：行为间通过事件总线通信，避免直接耦合。
- **FR-7**：当用户在宠物身上按下鼠标并移动时，立即进入 `Drag`，优先级最高。
- **FR-8**：松开拖拽后根据释放速度决定进入 `Idle` 或 `Fall`。
- **FR-9**：`AutoScheduler` 在无交互时按可配置权重切换 `Idle` / `Walk` / `Jump`。
- **FR-10**：单击宠物 → 反馈动画；双击 → 跳跃；右键 → 上下文菜单（退出 / 临时隐藏 / 设置）。
- **FR-11**：宠物碰到屏幕左右边缘时执行转身或反弹策略。
- **FR-12**：用户配置通过 `ConfigStore` 持久化到 `config.json`。
- **FR-13**：应用启动后默认隐藏到系统托盘，托盘菜单提供"显示宠物 / 退出"。
- **FR-14**：所有公开 API 用 TypeScript 声明，必要单元测试随实现一同提交。

---

## 5. Non-Goals (Out of Scope)

以下内容**不属于** v1.0 范围：

- 多只宠物同屏（仅支持单只）
- 宠物之间或与其他用户的网络互动 / 联机功能
- 自定义美术资源导入 / 皮肤商店
- 复杂 AI（语音对话、LLM 接入、情绪系统等）
- 喂食、清洁、生命值、成长系统等养成机制
- 计时器 / 番茄钟 / 日程提醒等"工作助手"功能
- 移动端、Linux 端适配（仅 Windows + macOS）
- 自动更新机制（手动安装即可）
- 安装包签名与公证流程

可作为后续版本通过插件式行为接口扩展，但不进入 v1.0。

## 6. Design Considerations

- **视觉风格**：圆润、低饱和、亲和力强；避免锐利边缘和刺眼颜色。
- **动画**：所有动作动画时长 ≤ 1.5s，避免长时间锁住交互响应。
- **窗口策略**：使用全屏透明窗口 + 鼠标穿透，仅在宠物精灵命中区域接收点击。
- **资源组织**：动画资源按行为目录组织（如 `/assets/idle/`、`/assets/walk/`），便于后续替换。
- **可访问性**：右键菜单文字遵循系统语言；提供"减少动画"选项以照顾对动画敏感的用户（v1.0 至少预留配置项）。

---

## 7. Technical Considerations

- **技术栈**：Electron、TypeScript、Vite/Webpack 任一构建工具、Vitest/Jest 任一测试框架。
- **架构分层**：
  - `core/`：状态机、事件总线、配置中心、调度器
  - `behaviors/`：每个行为一个文件夹，包含逻辑 + 动画资源 + 测试
  - `render/`：动画渲染层（SVG/Lottie 适配器）
  - `platform/`：Electron 主进程、屏幕信息、托盘等系统集成
- **窗口透明 + 鼠标穿透**：使用 `setIgnoreMouseEvents(true, { forward: true })`，仅在精灵命中区切回 `false`。
- **多屏支持**：v1.0 仅强保证主屏；多屏边界检测可放在 v1.1。
- **依赖最小化**：除 Electron 本体外，运行时依赖控制在 5 个以内，降低体积与攻击面。

---

## 8. Success Metrics

- 5 个核心行为（`Idle` / `Walk` / `Jump` / `Drag` / `Fall`）全部实现并可稳定切换。
- 在中端机器上空闲 CPU < 2%，内存 < 200 MB。
- 用户可在 1 次操作内（拖拽）将宠物移动到屏幕任意位置。
- 新增一个示例行为（如 `SitBehavior`）所需改动 ≤ 1 个新文件 + 1 行注册代码，验证插件式架构有效。
- 连续运行 8 小时无崩溃。
